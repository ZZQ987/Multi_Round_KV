# 修改transformers 库实现多轮多模态（含图片文字）对话

## 主要的修改点有三处：

对于原来的实现：
1. 有cache 情况下，无法处理新的视觉输入，会直接drop
2. 有cache 情况下，rope 的计算只支持后续纯文本输入，无法进行MRoPE计算
3. 重复编码所有的视觉输入 

对以上情况进行了修正

## 特别需要注意：

为了cache 位置的准确，多轮对话拼接输入输出时，不要引入额外的空白和换行，这会导致cache 的错位

## 未解决的问题（非代码逻辑错误）：
我认为是底层实现的问题：
1. 同样的视频/图片，在单独输入和与其他图片一起输入ViT时，可能得到不同的特征向量
2. 假设采用原来的默认实现（也就是只支持文本cache），每轮都要重新prefill，此时，前面区域某些位置的kv cache会改变

以上两个问题会轻微影响benchmark 的点数。
其中第二个问题更明显一点，对于qwen3vl 也有类似的问题
